# Accessibility status

Target: WCAG 2.1 AA. Last reviewed after stage_polish.

## In place

- **Skip-to-content link** is the first focusable element of `(dashboard)/layout.tsx`. It targets `#main-content` (the `<main>` inside `AppShell`).
- **Global focus ring**: `:focus-visible { ring }` in `globals.css` so every focusable element shows a visible ring; never overridden with `outline-none` without a replacement.
- **Icon-only buttons** carry `aria-label` (sidebar menu toggle, user menu, action icons on `/repeat-orders`, etc.).
- **Status colors paired with text/icon**: `StatusBadge` always renders a textual label alongside the color.
- **Form labels** wire to inputs via `<Label htmlFor>` and `id`.
- **Toaster** (Sonner) sets `aria-live="polite"` by default.
- **Focus traps in dialog/sheet**: shadcn primitives based on Radix include focus management; Esc to close, focus returns to trigger.
- **Keyboard shortcuts** include `?` (cheat sheet), `g u/r/q/s` (route nav), and `/` (focus the page's primary search input).

## Known not-fixed-yet

- **Automated axe audit not wired into CI** — verified manually with axe DevTools on `/repeat-orders`, `/uploads`, `/settings`, and `/review-requests` after stage_polish; zero violations at that time. Not regression-protected; add an axe-core Playwright check in a future pass.
- **Screen reader smoke test** done with VoiceOver on `/login` → `/dashboard` → `/repeat-orders`. The status badges read out correctly thanks to the textual labels. The row-level action icon names are announced via tooltip+aria-label.
- **Color contrast** spot-checked on the indigo primary against background/surfaces. Pass-rate ≥ 4.5:1 on body text; the `muted-foreground` token is the contrast floor — verify if you adjust the palette.
